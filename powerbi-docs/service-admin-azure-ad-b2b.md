# <a name="distribute-power-bi-content-to-external-guest-users-with-azure-ad-b2b"></a>Предоставление содержимого Power BI внешним гостевым пользователям с помощью Azure AD B2B

Power BI интегрируется с Azure Active Directory B2B (Azure AD B2B), чтобы обеспечить безопасное распространение содержимого Power BI для гостевых пользователей за пределами организации и сохранить контроль над внутренними данными.  

Кроме того вы можете разрешить гостевым пользователям за пределами вашей организации редактировать содержимое в вашей организации и управлять им.

## <a name="enable-access"></a>Предоставление доступа

Не забудьте включить [использовать содержимое совместно с внешними пользователями](service-admin-portal.md#export-and-sharing-settings) функции на портале администрирования Power BI, прежде чем приглашать гостевых пользователей.

Можно также использовать [разрешить внешние пользователи-Гости для редактирования и управления содержимым в организации](service-admin-portal.md#allow-external-guest-users-to-edit-and-manage-content-in-the-organization) функции. Он позволяет выбрать, какие гостевого пользователя можно просматривать и создавать содержимое в рабочих областях, включая обзор Power BI в организации.

## <a name="who-can-you-invite"></a>Кого вы можете пригласить?

Вы можете приглашать гостевых пользователей с помощью любого адреса электронной почты, включая личные учетные записи, такие как gmail.com и outlook.com, hotmail.com. Azure AD B2B вызывает эти адреса *удостоверений социальных сетей*.

## <a name="invite-guest-users"></a>Приглашение гостевых пользователей

Гостевым пользователям требуется только приглашения первый раз, пригласить его в вашей организации. Существует два способа для приглашения пользователей: запланированные приглашения и нерегламентированных приглашения.

### <a name="planned-invites"></a>Плановые приглашения

Используйте плановое приглашение, если знаете, каких пользователей нужно пригласить. Вы можете отправлять приглашения с помощью портала Azure или PowerShell. Для этого вам потребуются права администратора клиента.

Чтобы отправить приглашение на портале Azure, выполните следующие действия.

1. На [портале Azure](https://portal.azure.com)выберите **Azure Active Directory**.

1. В разделе **управление**выберите **пользователей** > **всех пользователей** > **новый гостевой пользователь**.

    ![Снимок экрана: портал Azure, используя новый параметр гостевой пользователь вызваны.](media/service-admin-azure-ad-b2b/azure-ad-portal-new-guest-user.png)

1. Введите **адрес электронной почты** и **личное сообщение**.

    ![Снимок экрана диалогового окна Azure AD портал новый гостевой пользователь.](media/service-admin-azure-ad-b2b/azure-ad-portal-invite-message.png)

1. Выберите **Пригласить**.

Чтобы пригласить несколько гостевых пользователей, воспользуйтесь PowerShell. Дополнительные сведения см. в статье [Примеры кода и команд PowerShell для службы совместной работы Azure Active Directory B2B](/azure/active-directory/b2b/code-samples/).

В полученном электронном письме с приглашением гостевому пользователю нужно выбрать **Get Started** (Приступить к работе). Затем гостевой пользователь добавляется к клиенту.

![Снимок экрана для гостевых пользователей по электронной почте приглашение.](media/service-admin-azure-ad-b2b/guest-user-invite-email.png)

### <a name="ad-hoc-invites"></a>Ad hoc приглашения

Чтобы пригласить внешнего пользователя в любое время, добавьте их к панели мониторинга или отчет через общий пользовательский Интерфейс или приложения с помощью страницы доступа. Ниже приведен пример того, что нужно сделать, чтобы пригласить внешнего пользователя для работы с приложением.

![Снимок экрана внешний пользователь добавлен в список доступа приложения в Power BI.](media/service-admin-azure-ad-b2b/power-bi-app-access.png)

Гостевой пользователь получит электронное сообщение, указывающее, общий доступ к приложению с ними.

![Снимок экрана из электронной почты для приложения, совместно с гостевой пользователь](media/service-admin-azure-ad-b2b/guest-user-invite-email-2.png)

Гостевой пользователь должен войти с использованием организационного адреса электронной почты. Они получат запрос на принятие приглашения после входа в систему. После входа в приложение откроется для гостевого пользователя. Чтобы вернуться в приложение, пользователь может добавить ссылку в закладки или сохранить письмо.

## <a name="licensing"></a>Лицензирование

Гостевой пользователь должен иметь соответствующие требования лицензирования для просмотра содержимого, к которому вы предоставили доступ. Существует три способа, чтобы убедиться, что пользователь имеет лицензии: используйте Power BI Premium, назначить лицензии Power BI Pro или лицензия Power BI Pro гостевого пользователя.

При использовании функции [Разрешение внешним гостям отслеживать и изменять содержимое в организации](service-admin-portal.md#allow-external-guest-users-to-edit-and-manage-content-in-the-organization) гостевым пользователям, которые вносят содержимое в рабочие области или делятся им с другими пользователями, нужна лицензия Power BI Pro.

### <a name="use-power-bi-premium"></a>Использование Power BI Premium

Назначение рабочей области приложения для [емкости Power BI Premium](service-premium-what-is.md) позволяет использовать приложение не требуется лицензия Power BI Pro гостевой пользователь. Power BI Premium также позволяет использовать преимущества других возможностей, таких как частые обновления, выделенные ресурсы и крупные модели приложения.

![Схема гостевой взаимодействие с пользователем с помощью Power BI Premium.](media/service-admin-azure-ad-b2b/license-approach-1.png)

### <a name="assign-a-power-bi-pro-license-to-guest-user"></a>Назначение гостевому пользователю лицензии Power BI Pro

Назначение лицензий Power BI Pro гостевого пользователя в своем клиенте позволяет содержимого представления гостевого пользователя в клиенте.

![Схема гостевой взаимодействие с пользователем с лицензией назначить Pro из вашего клиента.](media/service-admin-azure-ad-b2b/license-approach-2.png)

### <a name="guest-user-brings-their-own-power-bi-pro-license"></a>Применение лицензии Power BI Pro гостевого пользователя

Гостевому пользователю уже назначена лицензия Power BI Pro в клиенте.

![Схема гостевой взаимодействие с пользователем при их переносить свои собственные лицензии.](media/service-admin-azure-ad-b2b/license-approach-3.png)

## <a name="guest-users-who-can-edit-and-manage-content"></a>Гостевые пользователи, которые могут редактировать содержимое и управлять им 

При использовании [разрешить внешние пользователи-Гости для редактирования и управления содержимым в организации](service-admin-portal.md#allow-external-guest-users-to-edit-and-manage-content-in-the-organization) компонента, указанного гостевых пользователей получить доступ к Power BI в организации. Они могут видеть все содержимое, к которому требуется разрешение. Они могут Главная доступ, Обзор рабочих областей, устанавливать приложения, см. в разделе, где они находятся в списке доступа к и свой вклад в содержимое в рабочих областях. Они могут создавать или администрировать рабочие области, использующие новые возможности взаимодействия. Действуют некоторые ограничения. В разделе рекомендации и ограничения перечислены эти ограничения.
 
Для этих пользователей войти в Power BI, предоставьте URL-адрес клиента. Чтобы найти URL-адрес клиента, выполните указанные ниже действия.

1. В службе Power BI в верхнем меню выберите справку ( **?** ) и **О Power BI**.

2. Найдите значение рядом с полем **URL-адрес клиента**. Значение URL-адрес клиента, вы можете предоставить гостевым пользователям.

    ![Снимок экрана о Power BI диалоговое окно с клиента гостевого пользователя, вызванной URL-адрес.](media/service-admin-azure-ad-b2b/power-bi-about-dialog.png)

## <a name="considerations-and-limitations"></a>Рекомендации и ограничения

* По умолчанию внешние Azure AD B2B ограничивает гостей использовать только содержимое. Внешние гостевые пользователи Azure AD B2B можно просмотреть приложения, панели мониторинга, отчеты, экспортировать данные и создавать подписки электронной почты для панелей мониторинга и отчетов. Они не могут использовать рабочие области и публиковать собственное содержимое. Тем не менее, эти ограничения не применяются для гостевых пользователей, которые получают доступ через [разрешить внешние пользователи-Гости для редактирования и управления содержимым в организации](service-admin-portal.md#allow-external-guest-users-to-edit-and-manage-content-in-the-organization) функции.

* Для гостевых пользователей, выполненной [разрешить внешние пользователи-Гости для редактирования и управления содержимым в организации](service-admin-portal.md#allow-external-guest-users-to-edit-and-manage-content-in-the-organization) компонентов, некоторые возможности недоступны на них. Чтобы изменять или публиковать отчеты, им нужно использовать пользовательский веб-интерфейс службы Power BI, включая функцию "Получить данные" для отправки файлов Power BI Desktop.  Не поддерживаются следующие возможности:
    * Прямая публикация из Power BI Desktop в службу Power BI.
    * Гостевые пользователи не могут использовать Power BI desktop для подключения к наборам данных службы в службе Power BI
    * Классические рабочие области привязаны к Группам Office 365:
        * Гостевой пользователь не может создать или "Администраторы" этих рабочих областей
        * Гостевые пользователи могут быть участниками
    * Отправка нерегламентированных приглашения не поддерживается для списков доступа к рабочей области
    * Power BI Publisher для Excel не поддерживается для гостевых пользователей
    * Гостевые пользователи нельзя установить Power BI Gateway и подключить его к вашей организации
    * Гостевые пользователи не могут устанавливать для всей организации публикация приложений
    * Гостевые пользователи не может использовать, создать, обновить или установить пакеты содержимого организации
    * Гостевые пользователи не могут использовать анализ в Excel
    * Гостевые пользователи не могут быть @mentioned в комментарии
    * Гостевые пользователи не могут использовать подписки
    * Гостевые пользователи, использующие эту возможность, должны иметь рабочую или учебную учетную запись. С помощью личных учетных записей гостевых пользователей могут возникнуть дополнительные ограничения из-за должны входить в ограничения.

* Эта функция в настоящее время недоступна для веб-части отчетов Power BI SharePoint Online.

* Существуют параметры Active Directory, можно ограничить возможности внешних гостевых пользователей в организации в целом. Также относится к вашей среде Power BI. Эти параметры описаны в следующей документации:
    * [Управление параметрами внешнего взаимодействия](https://docs.microsoft.com/azure/active-directory/b2b/delegate-invitations#control-who-can-invite)
    * [Разрешение или запрет приглашений для пользователей B2B из определенных организаций](https://docs.microsoft.com/azure/active-directory/b2b/allow-deny-list)
