# <a name="use-auditing-within-your-organization"></a>Используйте аудита в своей организации

Зная, какие элементы в Power BI, занимает какое действие клиента может быть вызвана в успехе вашей организации выполнения требований, таких как соответствия нормативным требованиям и управления записями. Используйте Power BI аудита для аудита действий, сделанные пользователями, например «Просмотр отчета» и «Просмотр панели мониторинга». Не удается использовать аудит для аудит разрешений.

Использовать функцию аудита можно в Центре безопасности и соответствия требованиям Microsoft Office 365 или с помощью PowerShell. Аудит зависит от функциональных возможностей Exchange Online (автоматически подготавливается для Power BI).

Можно фильтровать данные по диапазон дат, пользователю, панели мониторинга, отчетов, набор данных и тип действия аудита. Вы также можете скачать действия в файл csv (значения с разделителями запятыми) для анализа в автономном режиме.

## <a name="requirements"></a>Требования

Чтобы получить доступ к журналам аудита, вы должны удовлетворить следующие требования:

* Должен быть глобальным администратором, либо ролью журналы аудита или журналы аудита ящику в Exchange Online для доступа к журналу аудита. По умолчанию группы ролей управления соответствием и управление организацией в состав роли, назначенные на **разрешения** страницы в центре администрирования Exchange.

    Чтобы предоставить учетным записям без прав администратора доступ к журналу аудита, необходимо добавить пользователя членом одной из этих групп ролей. Если вы хотите сделать это другим способом, можно создать группу настраиваемой роли в центре администрирования Exchange, назначьте роль журналы аудита или журналами аудита ящику в эту группу и затем добавьте учетную запись без прав администратора в новую группу ролей. См. дополнительные сведения об [управлении группами ролей в Exchange Online](/Exchange/permissions-exo/role-groups).

    Если у вас нет доступа к Центру администрирования Exchange из Центра администрирования Microsoft 365, перейдите на страницу https://outlook.office365.com/ecp и войдите в систему с помощью своих учетных данных.

* Если вы имеют доступ к журналу аудита, но не глобальным администратором или администратором службы Power BI, не предоставляется доступ к порталу администрирования Power BI. В этом случае вам нужно использовать прямую ссылку на [Центр безопасности и соответствия требованиям Office 365](https://sip.protection.office.com/#/unifiedauditlog).

## <a name="access-your-audit-logs"></a>Доступ к журналам аудита

Для доступа к журналам, сначала убедитесь, что для включения ведения журналов в Power BI. Дополнительные сведения см. в разделе [Журналы аудита](service-admin-portal.md#audit-logs) в документации на портале администрирования. Может быть до 48 часов задержки между моментом, когда аудит включен и при просмотре данных аудита. Если данные не отображаются сразу, проверьте журналы аудита позже. Аналогичная задержка возможна при получении разрешения на просмотр журналов аудита: доступ к ним может быть предоставлен не сразу.

Журналы аудита Power BI доступны непосредственно из [Центра безопасности и соответствия требованиям Office 365](https://sip.protection.office.com/#/unifiedauditlog). Есть также ссылка на портале администрирования Power BI:

1. В Power BI, выберите **значок шестеренки** в правом верхнем углу, затем выберите **портал администрирования**.

   ![Снимок экрана выберите в раскрывающемся меню шестеренки с параметром портала администрирования вызваны.](media/service-admin-auditing/powerbi-admin.png)

1. Выберите **Журналы аудита**.

1. Нажмите кнопку **Перейти в Центр администрирования Office 365**.

   ![Снимок экрана портала администрирования с помощью аудита записывает параметр и дороге приведены параметры Центр администрирования Microsoft Office 365.](media/service-admin-auditing/audit-log-o365-admin-center.png)

## <a name="search-only-power-bi-activities"></a>Поиск только действий Power BI

Ограничьте результаты поиска только действиями Power BI, выполнив следующие действия. См. дополнительные сведения о [списке действий, проверяемых Power BI](#activities-audited-by-power-bi) далее в этой статье.

1. На **поиска по журналам аудита** раздела **поиска**, выберите в раскрывающемся списке для **действия**.

2. Выберите в нем пункт **Действия Power BI**.

   ![Поиск по журналам с действиями Power BI, вызванной снимок экрана для аудита.](media/service-admin-auditing/audit-log-search-filter-by-powerbi.png)

3. Щелкните в любом месте за пределами поля выбора, чтобы закрыть список.

Результаты поиска будут возвращены только действий Power BI.

## <a name="search-the-audit-logs-by-date"></a>Поиск журналов аудита по дате

С помощью полей **Дата начала** и **Дата окончания** можно искать журналы по диапазону дат. По умолчанию выбирается за последние семь дней. Отображение обеспечивает дату и время в формате UTC (UTC). Максимально возможный диапазон дат составляет 90 дней. 

Вы получите ошибку, если выбранный диапазон дат превышает 90 дней. Если используется максимальный диапазон в 90 дней, выберите текущее время для поля **Дата начала**. В противном случае отобразится сообщение об ошибке, предупреждающее, что дата начала предшествует дате окончания. Если функция аудита была включена в течение последних 90 дней, начало диапазона дат не может предшествовать дате включения этой функции.

![Поиск по журналам аудита из экрана с параметрами даты начала и окончания, вызванной.](media/service-admin-auditing/search-audit-log-by-date.png)

## <a name="search-the-audit-logs-by-users"></a>Поиск журналов аудита по пользователям

Можно искать записи журнала аудита для действий, сделанных конкретным пользователям. Введите один или несколько имен пользователей в **пользователей** поля. Имя пользователя выглядит как адрес электронной почты. Это учетная запись, в которой пользователи вошли в Power BI с. Чтобы поиск вернул записи для всех пользователей (и учетных записей служб) в организации, оставьте это поле пустым.

![Поиск по пользователям](media/service-admin-auditing/search-audit-log-by-user.png)

## <a name="view-search-results"></a>Просмотр результатов поиска

После выбора **поиска**, загрузить результаты поиска. Через несколько секунд, они будут отображаться в **результаты**. По завершении поиска отображаются число найденных результатов. **Поиск по журналам аудита** может содержать не более 1000 событий. Если условиям поиска соответствует более чем на 1000 событий, приложение отображает последние 1000 событий.

### <a name="view-the-main-results"></a>Просмотр основных результатов

**Результаты** область содержит следующие сведения для каждого события, возвращенных в результатах поиска. Чтобы отсортировать результаты, щелкните заголовок столбца в области **Результаты**.

| **Столбец** | **Определение** |
| --- | --- |
| Даты |Дата и время (в формате UTC) события. |
| IP-адрес |IP-адрес устройства, используемого для журнала действий. В приложении отображается IP-адрес в формате адреса IPv4 или IPv6. |
| Пользователь |Пользователь (или учетная запись службы), выполнивший действие, которое вызвало событие. |
| Activity |Действие, выполненное пользователем. Это значение соответствует действиям, выбранным в раскрывающемся списке **Действия**. Для события из журнала аудита администратора Exchange значением в этом столбце является командлет Exchange. |
| Элемент |Объект создан или изменен из-за соответствующее действие. Например, учетной записи пользователя, обновленные или просматриваемого или измененного файла. Значения в этом столбце отображаются не для всех действий. |
| Подробности |Дополнительные сведения о действии. Опять же не все действия имеют значение. |

### <a name="view-the-details-for-an-event"></a>Просмотр сведений о событии

Чтобы просмотреть дополнительные сведения о событии, выберите запись событий в списке результатов поиска. Объект **сведения** откроется страница с подробными свойствами из записи события. **Сведения** страницы отображаются свойства зависимости от службы Office 365, в котором происходит событие.

Для отображения этих сведений выберите **Дополнительные сведения**. Свойство RecordType всех записей в Power BI имеет значение 20. См. дополнительные сведения о [свойствах в журнале аудита](/office365/securitycompliance/detailed-properties-in-the-office-365-audit-log/).

   ![Снимок экрана с параметром Дополнительные сведения приведены в диалоговом окне сведений аудита.](media/service-admin-auditing/audit-details.png)

## <a name="use-powershell-to-search-audit-logs"></a>Поиск в журналах аудита с помощью PowerShell

Вы также можете получить доступ к журналам аудита по имени пользователя с помощью PowerShell. В следующем примере показано, как подключиться к Exchange Online PowerShell и извлечь записи журнала аудита Power BI с помощью команды [Search-UnifiedAuditLog](/powershell/module/exchange/policy-and-compliance-audit/search-unifiedauditlog?view=exchange-ps/). Для запуска скрипта, администратор должен назначить вам соответствующие разрешения, как описано в разделе [требования](#requirements) раздел.

```powershell
Set-ExecutionPolicy RemoteSigned

$UserCredential = Get-Credential

$Session = New-PSSession -ConfigurationName Microsoft.Exchange -ConnectionUri https://outlook.office365.com/powershell-liveid/ -Credential $UserCredential -Authentication Basic -AllowRedirection

Import-PSSession $Session
Search-UnifiedAuditLog -StartDate 9/11/2018 -EndDate 9/15/2018 -RecordType PowerBI -ResultSize 1000 | Format-Table | More
```

Дополнительные сведения о подключении к Exchange Online см. в разделе [Подключение к Exchange Online с помощью удаленной оболочки PowerShell](/powershell/exchange/exchange-online/connect-to-exchange-online-powershell/connect-to-exchange-online-powershell/). Еще один пример использования PowerShell с журналами аудита описан в руководстве по [использованию журнала аудита Power BI и PowerShell для назначения лицензий Power BI Pro](https://powerbi.microsoft.com/blog/using-power-bi-audit-log-and-powershell-to-assign-power-bi-pro-licenses/).
