# <a name="configure-workloads-in-a-premium-capacity"></a>Настройка рабочих нагрузок в емкости Premium

Эта статья описывает включение и настройку рабочих нагрузок для емкостей Power BI Premium. По умолчанию емкости поддерживают только рабочие нагрузки, связанные с выполнением запросов Power BI. Вы можете включить и настроить дополнительные рабочие нагрузки для **[искусственного интеллекта (Cognitive Services)](service-cognitive-services.md)** , **[потоков данных](service-dataflows-overview.md#dataflow-capabilities-on-power-bi-premium)** и **[отчетов с разбивкой на страницы](paginated-reports-save-to-power-bi-service.md)** .

## <a name="default-memory-settings"></a>Параметры памяти по умолчанию

Рабочие нагрузки запросов оптимизированы для ресурсов, определенных SKU емкости Premium, и ограничены ими. Емкости Premium также поддерживают дополнительные рабочие нагрузки, способные использовать ресурсы емкости. Значения памяти по умолчанию для этих рабочих нагрузок основаны на узлах емкости, доступных для вашего номера SKU. Параметры максимального объема памяти не являются накопительными. Объем памяти до максимального указанного значения выделяется динамически для искусственного интеллекта и потоков данных, но статически для отчетов с разбивкой на страницы.

## <a name="workload-settings"></a>Параметры рабочей нагрузки

### <a name="ai-preview"></a>AI (Предварительная версия)

В дополнение к **максимальный объем памяти** параметра рабочей нагрузки искусственного Интеллекта есть дополнительный параметр, **разрешить использование из Power BI Desktop**. По умолчанию используется **Off**. Этот параметр зарезервирован для будущего использования и может не отображаться на всех клиентах.

### <a name="datasets-preview"></a>Наборы данных (Предварительная версия)

По умолчанию наборы данных рабочей нагрузки включен и не может быть отключено. Эта рабочая нагрузка содержит дополнительный параметр, **конечной точки XMLA**. По умолчанию используется **1**, значение включено. Этот параметр указывает, что задать на уровне рабочей области и приложение членство в группе безопасности соблюдать подключения из клиентских приложений. Дополнительные сведения см. в разделе [подключение к наборам данных с помощью клиентских приложений и средств](service-premium-connect-tools.md).

### <a name="dataflows"></a>Потоки данных

В дополнение к **максимальный объем памяти** параметра рабочей нагрузки потоки данных есть дополнительный параметр, **размер контейнера**. Этот параметр позволяет оптимизировать производительность рабочей нагрузки потока данных для обработки потоков данных более сложных, большим количеством вычислений.

При обновлении потока данных, рабочая нагрузка потока данных создает контейнер для каждой сущности в потоке данных. Каждый контейнер может получить память до том в, указанного в параметре размер контейнера. Значение по умолчанию для всех номеров SKU — **700 МБ**. Может потребоваться изменить этот параметр, если:

- Потоки данных слишком долго для обновления или обновления потока данных завершается сбоем по истечении времени ожидания.
- Объекты потока данных включают шаги вычисления, например, соединение.  

Он, рекомендуем вам использовать [метрики емкости Power BI Premium](service-admin-premium-monitor-capacity.md) приложения для анализа производительности рабочей нагрузки потока данных. 

В некоторых случаях увеличив размер контейнера может повысить производительность. Например если поток данных получает данные только из источника без значительных вычислениях, изменив размер контейнера, возможно, не поможет. Увеличив размер контейнера может помочь, если она открывает целый рабочей нагрузки потока данных выделить дополнительную память для сущности операции обновления. Благодаря наличию больше памяти выделяется, он может сократить время, необходимое для обновления сущностей во многом вычисляемый. 

Значение размера контейнера не может превышать максимальный объем памяти для рабочей нагрузки потоков данных. Например емкость P1 имеет 25 ГБ памяти. Если рабочая нагрузка потока данных максимальный объем памяти (%) имеет значение 20% размера контейнера (МБ) не может превышать 5000. В любом случае размер контейнера не может превышать максимальный объем памяти, даже если задать более высокое значение. 

### <a name="paginated-reports-preview"></a>Отчеты с разбивкой на страницы (Предварительная версия)

Отчеты с разбивкой на страницы разрешить пользовательский код для выполнения при создании отчета. Например динамическое изменение цвета текста на основе содержимого, который может занять дополнительную память. Power BI Premium выполняет отчеты с разбивкой на страницы в автономной области в пределах емкости. Максимальный объем памяти, используется параметр *ли* рабочей нагрузки является активным. Если изменение параметра максимальный объем памяти по умолчанию, убедитесь, что выбрано низкое достаточно, чтобы он не влияет на отрицательно других рабочих нагрузок.

В некоторых случаях рабочая нагрузка отчетов с разбиением на страницы могут стать недоступными. В этом случае рабочая нагрузка показывает состояние ошибки на портале администрирования, и пользователи видят времени ожидания для подготовки отчета к просмотру. Чтобы устранить эту проблему, отключите рабочей нагрузки и затем включите ее повторно.

## <a name="configure-workloads"></a>Настройка рабочих нагрузок

Вы можете максимально увеличить емкость доступных ресурсов, включая рабочие нагрузки, только если они будут использоваться. Изменяйте параметры, только если параметры по умолчанию не соответствуют требованиям к ресурса емкости.  

### <a name="to-configure-workloads-in-the-power-bi-admin-portal"></a>Настройка рабочих нагрузок на портале администрирования Power BI

1. Выберите емкость в разделе **Параметры емкости** > **Емкости Premium**.

1. В разделе **Дополнительно** разверните **Рабочие нагрузки**.

1. Включите одну или несколько рабочих нагрузок и задайте значение для параметра **Максимальный объем памяти**.   

    
    ![Включение рабочих нагрузок](media/service-admin-premium-workloads/admin-portal-workloads.png)

1. Щелкните **Применить**.

### <a name="rest-api"></a>API-интерфейсы REST

Рабочие нагрузки можно включить и назначить емкости с помощью REST API [емкостей](https://docs.microsoft.com/rest/api/power-bi/capacities).
