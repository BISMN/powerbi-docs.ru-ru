# <a name="organize-work-in-the-new-workspaces-in-power-bi"></a>Организовать работу в новые рабочие области в Power BI

 *Рабочие области* — где совместно работать с коллегами для создания коллекции панелей мониторинга, отчеты и отчеты с разбивкой на страницы. Новые возможности рабочей области помогает лучше управлять доступом к содержимому. В этой статье описываются новые рабочие области, и чем они отличаются от классической рабочих областей.  Как с классической рабочими областями, вы по-прежнему использовать их для создания и распространения приложений. Читать о том, как [создать новые возможности рабочей области](service-create-the-new-workspaces.md).

Новые возможности рабочей области вышла общедоступная (GA) и теперь рабочую область по умолчанию. Вы по-прежнему можете создавать и использовать [классической рабочие области](service-create-workspaces.md) на основе групп Office 365. 

> [!NOTE]
> Чтобы принудительно использовать безопасность на уровне строк (RLS) для пользователей Power BI Pro, просмотра содержимого в рабочей области, продолжают использовать [классической рабочие области](service-create-workspaces.md). Выберите **участники могут только просматривать содержимое Power BI** параметр. Кроме того публикация приложений Power BI для этих пользователей или совместно использующих распространения содержимого. Готовящегося роль наблюдателя будет реализовать этот сценарий в будущем, в новые рабочие области качества рабочей области.

С помощью новых рабочих областей вы можете:

- Назначение ролей рабочей области для групп пользователей: группы безопасности, списки рассылки, группы Office 365 и отдельные пользователи.
- Создание рабочей области в Power BI без создания группы Office 365.
- Используйте более детализированные роли рабочих областей для более гибкого управления разрешениями в рабочей области.

При создании новой рабочей области вы не создаете соответствующую базовую группу Office 365. Все действия по администрированию рабочей области выполняются в Power BI, не в Office 365. В новом интерфейсе рабочей области теперь можно добавить группу Office 365 в списке доступа к рабочей области, чтобы продолжить, управление доступом пользователей к содержимому с помощью групп Office 365.

## <a name="administering-new-workspace-experience-workspaces"></a>Администрирование новые рабочие области качества рабочей области
Администрирование для новых рабочих областей возможности рабочей области — теперь в Power BI, выбрать Power BI "Администраторы" кто в организации могут создавать рабочие области. Они также можно управлять и восстановить рабочую область. Для этого им необходимо использовать портал администрирования Power BI или командлеты PowerShell. Для классической рабочих областей на основе групп Office 365 администрирования повторяется в портале администрирования Office 365 и Azure Active Directory.

В **параметры рабочей области** на портале администрирования, администраторы могут использовать создание рабочих областей (новый интерфейс рабочей области) параметр, чтобы разрешить все или никто в организации для создания новой рабочей области взаимодействие рабочих областей. Они также могут разрешить их создание только участникам определенных групп безопасности.

> [!NOTE]
> Создание рабочих областей (новый интерфейс рабочей области) установка значений по умолчанию, чтобы разрешить только пользователей, которые могут создавать группы Office 365 для создания новых рабочих областей в Power BI. Не забудьте установить значение на портале администрирования Power BI, чтобы убедиться, что соответствующие пользователи могут создавать новую рабочую область взаимодействие рабочих областей.

## <a name="roles-in-the-new-workspaces"></a>Роли в новых рабочих областях

Чтобы предоставить доступ к новой рабочей области, добавить группы пользователей или отдельных лиц в одну из ролей рабочей области: члены, участники или Администраторы. Каждый из пользователей в группе получает определенную вами роль. Если пользователь входит в несколько групп пользователей, они получают наивысший уровень разрешения, предоставляемые роли, которые они назначены.

Роли позволяют управлять разрешениями в рабочих областях для удобной совместной работы. В новых рабочих областях можно назначать роли пользователям и группам пользователей: группам безопасности, группам Office 365 и спискам рассылки. 

При назначении ролей группе пользователей у всех пользователей в группе будет доступ к содержимому. Если группа пользователей имеет вложения, разрешение получат все пользователи во вложенных группах. Если пользователь находится в нескольких группах пользователей с разными ролями, он получает самый высокий уровень разрешения. 

В новых рабочих областях есть три роли: администратор, член и участник.

|Возможность   | Администратор  | Член  | Участник  | 
|---|---|---|---|
| Обновлять и удалять рабочую область.  | X  |   |   |
| Добавлять и удалять пользователей, включая других администраторов.  | X  |   |   |
| Добавлять членов или других пользователей с разрешениями более низкого уровня.  |  X | X  |   |
| Публиковать и обновлять приложение. |  X | X  |   |
| Предоставлять общий доступ к элементу или приложению. |  X | X  |   |
| Разрешать другим пользователям повторно предоставлять совместный доступ к элементам. |  X | X  |   |
| Создавать, редактировать и удалять содержимое в рабочей области.  |  X | X  | X  |
| Публиковать отчеты в рабочей области, удалять содержимое. |  X | X  | X  |
 
 
## <a name="licensing"></a>Лицензирование
Каждый член рабочей области должен иметь лицензию Power BI Pro. Рабочая область позволяет этим пользователям совместно работать над панелями мониторинга и отчетами, которые вы планируете опубликовать для более широкой аудитории или даже для всей организации. Если вы хотите распространить содержимое для других пользователей в вашей организации, можно назначить им лицензии Power BI Pro или поместить рабочую область в емкость Power BI Premium.

> [!NOTE]
> Публикация отчетов на новый интерфейс рабочей области есть более строгие принудительного применения правила лицензирования. Пользователям, пытающимся публикация из Power BI Desktop или другими клиентами средств без лицензии Pro см. в разделе ошибку «только пользователи с лицензиями Power BI Pro можно опубликовать в эту рабочую область».

## <a name="how-are-the-new-workspaces-different-from-current-workspaces"></a>Чем отличаются новые рабочие области от существующих?

В новых рабочих областях мы изменяем некоторые функции. Ниже приведены изменения, которые можно ожидать постоянным. 

* Создание этих рабочих областей не будет создавать группы Office 365, как классические рабочих областей. Тем не менее теперь можно использовать группу Office 365 для предоставления пользователям доступа к рабочей области, назначив ей роли. 
* В классическом рабочих областях можно добавить только пользователи, элементов и список администраторов. В новых рабочих областях вы можете добавлять несколько групп безопасности AD, списков рассылки или групп Office 365 в эти списки для удобного управления пользователями. 
- Можно создать пакет содержимого организации из классической рабочей области. Нельзя создать такой пакет в новых рабочих областях.
- Вы можете использовать пакет содержимого организации из классической рабочей области. Нельзя использовать такой пакет в новых рабочих областях.

## <a name="workspace-contact-list"></a>Список контактов в рабочей области
Новый **список контактов** функция позволяет указать, какие пользователи получают уведомление о возникающих в рабочей области. По умолчанию любой пользователь или группа, определенные как рабочую область администратор получает уведомление, но его можно настроить. Пользователи или группы, перечисленные в списке контактов будет отображаться в интерфейсе пользователя (UI), чтобы помочь это пользователи получить помощь, относящиеся к рабочей области. 

Дополнительные сведения о [параметр списка контактов в рабочей области](service-create-the-new-workspaces.md#workspace-contact-list).

## <a name="workspace-onedrive"></a>OneDrive для рабочей области
Рабочая область OneDrive позволяет настроить группу Office 365, чье хранилище файл библиотеки документов SharePoint доступно для пользователей рабочей области. Группа должна быть создан за пределами Power BI. 

Power BI не будет синхронизировать разрешения пользователей или группы, которые имеют доступ к рабочей области с помощью членства в группах Office 365. Мы рекомендуем для управления доступом к рабочей области через Office 365 группу которого хранилище файлов, настройка этого параметра. 

Читать о том, как [задать и получить доступ к рабочей области OneDrive](service-create-the-new-workspaces.md#workspace-onedrive).

## <a name="limitations-and-considerations"></a>Рекомендации и ограничения

Учитывайте следующие ограничения:

- Рабочие области могут содержать не более 1000 наборов данных или 1000 отчетов на набор данных. 
- Пользователь с лицензией Power BI Pro может быть участником максимального 1000 рабочих областей.
- Power BI publisher для Excel не поддерживается.

## <a name="workspace-features-that-work-differently"></a>Функции рабочих областей, которые работают иначе

Некоторые функции работают по-разному в существующих и новых рабочих областях. Эти отличия являются намеренным, основываясь на отзывах, мы получили от клиентов и включить более гибкий способ совместной работы с рабочими областями:

- Применение лицензирования: Публикация отчетов на новый интерфейс рабочей области применяет существующие правила лицензирования, которым необходима лицензия Power BI Pro для пользователей, совместная работа в рабочих областях или совместном использовании содержимого другим пользователям в службе Power BI. Пользователи без лицензии Pro отображается ошибка «только пользователи с лицензиями Powre BI Pro можно опубликовать в эту рабочую область».
- Члены, которым можно или нельзя повторно делиться: заменено ролью участника.
- Рабочие области только для чтения: вместо предоставления пользователям доступа к рабочей области только для чтения вы сможете назначать пользователям готовящуюся к выходу роль читателя с аналогичными правами на чтение содержимого в рабочей области.
- Нет кнопки **Выход из рабочей области**.

## <a name="frequently-asked-questions"></a>Часто задаваемые вопросы

**Приводятся ссылки на существующее содержимое, затронутых новую рабочую область работы общедоступной версии**

Нет. Ссылки на существующие элементы в классическом рабочие области не влияет на новый интерфейс рабочей области. Общедоступная (GA) нового интерфейса рабочей области изменяется рабочую область по умолчанию, создать, но не изменения существующих рабочих областей. 

**Другие рабочие области, обновлены до нового интерфейса рабочей области с помощью GA**

Нет. Новые возможности глобальный Администратор рабочей области изменяется только тип рабочей области по умолчанию на новый интерфейс рабочей области. Существующих классической рабочих областей, которые основаны на группах Office 365 остаются неизменными.

**Рабочие области по-прежнему создаются автоматически для групп Office 365**

Да. Так как мы поддерживаем оба типа рабочих областей одновременно, мы продолжаем список всех групп Office 365, пользователь имеет доступ к в списке рабочих областей.
