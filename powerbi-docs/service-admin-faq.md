## <a name="sign-up-for-power-bi"></a>Регистрация в Power BI

### <a name="using-powershell"></a>Регистрация с помощью PowerShell

Для некоторых процедур, описанных в этом разделе, нужно использовать скрипты Windows PowerShell. Если вы не знакомы с PowerShell, рекомендуем прочесть статью [Начало работы с Windows PowerShell](http://go.microsoft.com/fwlink/p/?LinkID=286814). Для выполнения скриптов необходимо установить последнюю 64-разрядную версию [Azure Active Directory PowerShell для Graph](/powershell/azure/active-directory/).

### <a name="how-do-users-sign-up-for-power-bi"></a>Как пользователям зарегистрироваться в службе Power BI?

Как администратор, можно зарегистрироваться для Power BI через [веб-сайт Power BI](https://powerbi.microsoft.com) или [приобретают](https://admin.microsoft.com/AdminPortal/Home#/catalog) страницы центра администрирования Microsoft 365. Когда администратор регистрируется в Power BI, они можно назначить пользовательские лицензии для пользователей, которым требуется доступ.

Кроме того, отдельные пользователи в организации могут зарегистрироваться в Power BI через [веб-сайт Power BI](https://powerbi.microsoft.com). Когда пользователь в вашей организации регистрируется для Power BI, служба автоматически назначается лицензия Power BI пользователь. Дополнительные сведения см. в разделе [зарегистрироваться в Power BI в индивидуальном](service-self-service-signup-for-power-bi.md) и [Лицензирование Power BI в вашей организации](service-admin-licensing-organization.md).

### <a name="how-do-individual-users-in-my-organization-sign-up"></a>Как зарегистрироваться отдельным пользователям в организации?

Для пользователей в вашей организации возможны три сценария.

* **Сценарий 1.** В вашей организации уже используется среда Office 365 и у пользователя, регистрирующегося в Power BI, уже есть учетная запись Office 365.
    В этом случае если пользователь уже есть рабочая или учебная учетная запись в клиенте (например, contoso.com), но не еще нет Power BI, корпорация Майкрософт просто активирует план для этой учетной записи. Пользователь автоматически получает уведомление со сведениями о том, как использовать службу Power BI.

* **Сценарий 2.** В вашей организации есть среда Office 365, но у пользователя, регистрирующегося в Power BI, нет учетной записи Office 365.
    В этом случае пользователь имеет адрес электронной почты в домене вашей организации (например, contoso.com), но еще нет учетной записи Office 365. В этом случае пользователь может зарегистрироваться в Power BI и ему автоматически назначается учетная запись. Это действие позволяет пользователю доступ к службе Power BI. Например, если сотрудник по имени Юлия использует свой рабочий адрес электронной почты (например nancy@contoso.com) чтобы зарегистрироваться, корпорация Майкрософт автоматически добавляет Нэнси в качестве пользователя в среду Office 365 организации Contoso и активирует Power BI для этой учетной записи.

* **Сценарий 3.** Вашей организации нет среды Office 365, подключенной к вашему домену электронной почты.
    Существуют никаких административных действий, необходимых для вашей организации, чтобы воспользоваться преимуществами Power BI. Служба добавляет пользователей в каталог нового, доступного только для облачных пользователей. Вы также можете берет на себя обязанности администратора клиента и управлять ими.

> [!IMPORTANT]
> Если у вашей организации есть несколько доменов электронной почты и вы хотите, чтобы все расширения адресов размещались в одном клиенте, добавьте все эти домены в клиент Azure Active Directory до начала регистрации пользователей. После создания пользователей, то есть автоматизированная процедура перемещения пользователей по клиентам. Дополнительные сведения об этом процессе см. в разделе [Если у меня есть несколько доменов, управлять добавления пользователей в клиенте Office 365?](#if-i-have-multiple-domains-can-i-control-the-office-365-tenant-that-users-get-added-to) далее в этой статье и [Добавление домена к Office 365](/office365/admin/setup/add-domain/).

### <a name="how-can-i-prevent-users-from-joining-my-existing-office-365-tenant"></a>Как предотвратить присоединение пользователей к существующему клиенту Office 365?

Администратор может предпринять ряд действий, чтобы не позволить пользователям присоединиться к имеющемуся клиенту Office 365. Если заблокировать доступ пользователей попытки регистрации со стороны сбой, и появится сообщение, установленному значению обратитесь к администратору своей организации. Повторите эту процедуру, если ранее вы уже отключили автоматическую рассылку лицензий (например, с помощью Office 365 для образовательных учреждений для учащихся, преподавателей и сотрудников) не требуется.

Используйте приведенный ниже скрипт PowerShell, чтобы запретить новым пользователям присоединяться к управляемому клиенту. [См. дополнительные сведения о PowerShell][1].

```powershell
$msolcred = get-credential
connect-msolservice -credential $msolcred

Set-MsolCompanySettings -AllowEmailVerifiedUsers $false
```

> [!NOTE]
> После блокировки доступа новые пользователи в вашей организации не смогут зарегистрироваться в Power BI. Пользователи, которые зарегистрировались в Power BI до отключения новых регистраций, сохранят свои лицензии. Чтобы удалить такого пользователя, см. раздел [Как удалить доступ к Power BI для пользователей, которые уже зарегистрированы?](#how-do-i-remove-power-bi-for-users-that-already-signed-up) далее в этой статье.

### <a name="how-can-i-allow-users-to-join-my-existing-office-365-tenant"></a>Как разрешить пользователям присоединяться к существующему клиенту Office 365?

Используйте следующий сценарий PowerShell для новых пользователей присоединиться к управляемый клиент. [См. дополнительные сведения о PowerShell][1].

```powershell
$msolcred = get-credential
connect-msolservice -credential $msolcred

Set-MsolCompanySettings -AllowEmailVerifiedUsers $true
```

### <a name="how-do-i-check-if-i-have-the-block-on-in-the-tenant"></a>Как проверить наличие блока в клиенте?

Используйте следующий сценарий PowerShell для проверки параметров. Параметр *AllowEmailVerifiedUsers* должен иметь значение false. [См. дополнительные сведения о PowerShell][1].

```powershell
$msolcred = get-credential
connect-msolservice -credential $msolcred

Get-MsolCompanyInformation | fl allow*
```

### <a name="how-can-i-prevent-my-existing-users-from-starting-to-use-power-bi"></a>Как запретить имеющимся пользователям начать работу с Power BI?

В Azure AD за эту функцию отвечает параметр **AllowAdHocSubscriptions**. Большинство клиентов имеют это значение true, это означает, что он включен. Если вы приобрели Power BI через партнера, это может быть присвоено значение false, это означает, что она отключена.

Используйте приведенный ниже скрипт PowerShell, чтобы отключить нерегламентированные подписки. [См. дополнительные сведения о PowerShell][1].

1. Войдите в Azure Active Directory с использованием учетных данных Office 365. В первой строке следующего скрипта PowerShell запрашиваются учетные данные. Вторая строка используется для подключения к Azure Active Directory.

    ```powershell
     $msolcred = get-credential
     connect-msolservice -credential $msolcred
    ```

   ![Снимок экрана Azure Active Directory, вход с помощью PowerShell](media/service-admin-licensing-organization/azure-ad-sign-in.png)

1. После выполнения входа вы, выполните следующую команду, чтобы увидеть, как клиент уже настроен.

    ```powershell
     Get-MsolCompanyInformation | fl AllowAdHocSubscriptions
    ```

1. Выполните следующую команду, чтобы включить (`$true`) или отключить (`$false`) **AllowAdHocSubscriptions**.

    ```powershell
     Set-MsolCompanySettings -AllowAdHocSubscriptions $false
    ```

> [!NOTE]
> Используйте **AllowAdHocSubscriptions** флаг для управления некоторыми возможностями пользователей в вашей организации, включая возможность регистрироваться в службе управления правами Azure. Состояние этого флага влияет на все эти возможности.

### <a name="how-can-i-allow-my-existing-users-to-sign-up-for-power-bi"></a>Как разрешить существующим пользователям регистрироваться в Power BI?

Чтобы разрешить существующим пользователям зарегистрироваться в Power BI, выполните команду на предыдущий вопрос, но передать `$true` вместо `$false` на предыдущем шаге.

## <a name="administration-of-power-bi"></a>Администрирование Power BI

### <a name="how-will-this-change-the-way-i-manage-identities-for-users-in-my-organization-today"></a>Как изменится существующий в моей организации подход к управлению удостоверениями пользователей?

Для пользователей в вашей организации возможны три сценария.

* **Сценарий 1.** Если вашей организации уже есть среда Office 365, и все пользователи в вашей организации имеют учетные записи Office 365, нет изменений в порядок управления удостоверениями.

* **Сценарий 2.** Если вашей организации уже есть среда Office 365, но не все пользователи в вашей организации имеют учетные записи Office 365, мы создадим пользователя в клиенте и назначить лицензии с учетом пользователя рабочего или учебного адреса электронной почты.

    Таким образом число пользователей, которыми вы управляете в любой определенный момент по мере регистрации пользователей вашей организации для службы.

* **Сценарий 3.** Если у вашей организации нет среды Office 365, подключенной к вашему домену электронной почты, нет изменений в порядок управления удостоверениями.

    Служба добавляет пользователей в каталог нового, доступного только для облачных пользователей. Кроме того вы можете берет на себя обязанности администратора клиента и управлять ими.

### <a name="how-do-we-manage-power-bi"></a>Как управлять Power BI?

Power BI предоставляет портал администрирования, который позволяет просматривать статистику использования ссылки на Центр администрирования Microsoft 365 для управления пользователями и группами и предоставляет возможность управлять параметрами на уровне клиента.

Чтобы использовать портал администрирования Power BI, необходимо пометить вашу учетную запись как **глобального администратора** в Office 365 или Azure Active Directory, или кто-то необходимо назначить роль администратора службы Power BI с учетной записью пользователя. Дополнительные сведения см. в разделе [основные сведения о роли администратора Power BI](service-admin-role.md) и [портал администрирования Power BI](service-admin-portal.md).

### <a name="what-is-the-process-to-manage-a-tenant-created-by-microsoft-for-my-users"></a>В чем заключается процесс управления клиентом, созданным корпорацией Майкрософт для моих пользователей?

Когда пользователь самостоятельно регистрируется для облачной службы, использующей Azure AD, служба добавляет их в неуправляемых Azure AD directory на основании домена электронной почты. Вы можете утверждения и управлять клиентом, который кто-то создан с помощью процесса под названием *смены администратором*. Зависит от типа перенаправление, делается ли уже есть управляемый клиент, связанный с доменом:

* Используйте *внутреннюю смену* для создания управляемого клиента для домена.

* Используйте *внешнюю смену* для перемещения домена в существующий управляемый клиент.

Дополнительные сведения см. в разделе [перехватить неуправляемого каталога от имени администратора в Azure Active Directory](/azure/active-directory/users-groups-roles/domains-admin-takeover).

При этом внешнее перенаправление служба помещает Power BI содержимого, который был создан перед перенаправлением в [рабочая область Power BI архивной](service-admin-power-bi-archived-workspace.md). Вам нужно вручную переместить любое содержимое, которое вы хотите использовать, в новый клиент.

### <a name="if-i-have-multiple-domains-can-i-control-the-office-365-tenant-that-users-get-added-to"></a>Если у меня есть несколько доменов, управлять добавления пользователей в клиенте Office 365?

При отсутствии действий, служба создаст для каждого пользователя по электронной почте домена и поддомена клиент. Если вы хотите разместить всех пользователей в одном клиенте независимо от расширений адресов электронной почты, сделайте следующее: Создайте целевой клиент заранее или использовать существующий клиент. Затем добавьте все имеющиеся домены и поддомены, которые необходимо объединить в этом клиенте. Любой пользователь с адреса электронной почты, заканчивающиеся на этими доменами и поддоменами, автоматически присоединиться к целевому клиенту при регистрации.

> [!IMPORTANT]
> После создания пользователей, то есть автоматизированная процедура перемещения пользователей по клиентам. Дополнительные сведения о добавлении доменов в один клиент Office 365 см. в разделе [Добавление пользователей и доменов в Office 365](/office365/admin/setup/add-domain/).

### <a name="how-do-i-remove-power-bi-for-users-that-already-signed-up"></a>Как удалить доступ к Power BI для пользователей, которые уже зарегистрированы?

Если пользователь зарегистрировался в службе Power BI, но вы хотите запретить ему доступ к Power BI, можно удалить лицензию Power BI для этого пользователя.

1. Перейдите в [Центр администрирования Microsoft 365](https://admin.microsoft.com/AdminPortal/Home#/homepage).

1. В левой области навигации выберите **Пользователи** > **Активные пользователи**.

1. Найдите пользователя, лицензию которого нужно удалить, и выберите его имя.

    Также можно выполнить массовое управление лицензиями для пользователей. Для этого выберите несколько пользователей, а затем выберите **Изменение лицензий на продукты**.

1. В области сведений о пользователе, рядом с разделом **Лицензии на продукты** выберите **Изменить**.

1. Зависимости от того, что лицензии можно применить к своей учетной записью, задайте **Power BI (бесплатно)** или **Power BI Pro** для **Off**.

1. Нажмите кнопку **Сохранить**.

### <a name="how-do-i-know-when-new-users-have-joined-my-tenant"></a>Как узнать, что к моему клиенту подключились новые пользователи?

Пользователи, присоединившиеся к клиенту в рамках этой программы получить получают уникальную лицензию, можно выполнить фильтрацию по в своей области активного пользователя в панели мониторинга администратора. Чтобы создать представление, выполните описанные ниже действия.

1. Перейдите в [Центр администрирования Microsoft 365](https://admin.microsoft.com/AdminPortal/Home#/homepage).

1. В левой области навигации выберите **Пользователи** > **Активные пользователи**.

1. В меню **Представления** выберите **Добавление настраиваемого представления**.

1. Присвойте имя новому представлению и в разделе **Назначенная лицензия продукта** выберите пункт **Power BI (free)** (Power BI (бесплатная версия)) или **Power BI Pro**.

    Для одного представления можно выбрать только один тип лицензии. Если у организации есть лицензия на **Power BI (free)** (Power BI (бесплатная версия)) и на **Power BI Pro**, вы можете создать два представления.

1. Введите другие требуемые условия и выберите **Добавить**.

1. После создания нового представления оно становится доступным из **представления** меню.

### <a name="are-there-any-additional-things-i-should-prepare-for"></a>Существуют ли какие-нибудь подводные, следует подготовиться?

Вы можете удалить пользователя из клиента с помощью стандартной процедуры в Центре администрирования Microsoft 365. Но если у пользователя по-прежнему есть активный адрес электронной почты, предоставленный вашей организацией, он может снова присоединиться к клиенту, если только вы не заблокируете эту возможность для всех пользователей.
